## 三、目标联动机制

### 3.1 层级关系

```
总体推进计划（本文件）
    ↓ 拆解
年度目标（2026年.md）
    ↓ 拆解
月度目标（2026-05.md 等）
    ↓ 拆解
周计划（第1周.md 等）
    ↓ 拆解
每日任务（2026-05-21.md 等）
```

每一层必须能回答两个问题：
1. **这个目标从上一层哪个目标来的？**（向上追溯）
2. **完成了这个目标，下一层要做什么？**（向下分解）

### 3.2 推进节奏

| 节奏 | 做什么 | 由谁 |
|------|--------|------|
| **每日** | 执行当日任务 → 更新三线战报 + 执行日志 | 少爷执行，AI 记录 |
| **每周复盘**（周日/周一）| 检查周计划完成度 → 调整下周计划 → 更新推进看板 | AI 发起 |
| **月度复盘**（月底）| 检查月度目标 → 更新状态.json → 调整下月目标 → 推演下月风险 | AI 发起 |
| **阶段复盘**（阶段切换时）| 检查出口标准是否达成 → 更新年度目标 → 启动新阶段 | AI 发起 |

### 3.3 目标调整规则

| 场景 | 操作 |
|------|------|
| 月目标明显完不成（Week 2 就看出没戏）| 当周下调，不等到月底 |
| 某条线连续 3 天执行率 < 50% | 该线降级到最低版，直到恢复 |
| 意外收获（提前拿到 offer / 交易突然开窍）| 重新排优先级，但不盲目加量 |
| 外部冲击（生病 / 加班周 / 家庭事件）| 砍所有非必要线，只保身体 |

## 五、关键路径图

### 依赖关系

```
身体稳定
  ├─→ 有精力学交易
  ├─→ 有精力刷题
  └─→ 好的睡眠

交易学完 SMC
  ├─→ 模拟盘验证
  │    └─→ 微型实盘验证
  │         └─→ 实盘稳定盈利
  │              └─→ 评估是否转全职
  └─→ 行情笔记积累 → 交易直觉

工作刷题 + 简历
  ├─→ 面试
  │    └─→ 12k+ offer
  │         └─→ 月攒 7k+
  │              └─→ 10-14 万本金
  │                   └─→ 交易加速
  └─→ 面试积累 → 下次跳槽更顺
```

### 关键路径（最依赖链）

```
身体 → 精力 → [交易学习 + 刷题] → 交易/工作突破 → 攒钱 → 交易本金 → 自由
```

如果身体断了，一切暂停。所以**身体是整条链上优先级最高的节点**。

### 风险点

| 节点 | 风险 | 影响 | 备选方案 |
|------|------|------|----------|
| 身体中断 | 高 | 其他两条线也会跟着断 | 中断协议启动，最低版恢复 |
| 交易学不懂 | 中 | SMC 学不完，无法进入模拟盘 | 拉长学习周期，不赶进度 |
| 刷题不够 | 中 | 月 3 拿不到 offer | 推迟到 Month 4-5，或降低薪资要求 |
| 面试紧张 | 低 | 面试表现差 | 多面几次，每次复盘 |
