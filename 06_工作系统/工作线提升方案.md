# 工作线提升方案

> Java 后端 · 12-15k · 面试涨薪
> 刷题 + 简历 + 面试，三件事串成一条线

---

## 一、总览：三件事的节奏

```
Month 1（5/21~6/20）:  刷题打基础 + 简历定稿 + 确定目标公司
```

## 二、刷什么

### 2.1 刷题范围

Java 后端 12-15k 的面试，算法题不会太难。**高频题型就够了，不用碰冷门。**

| 优先级 | 题型 | 掌握程度 | 约多少题 |
|--------|------|----------|----------|
| P0 必刷 | 数组 + 字符串 + 哈希表 | 秒 Medium | 20-30 道 |
| P0 必刷 | 双指针 / 滑动窗口 | 秒 Medium | 10-15 道 |
| P0 必刷 | 二叉树（前中后层序） | 默写模板+变种 | 10-15 道 |
| P1 重点 | 二分查找 | 能写模板+找分界点 | 8-10 道 |
| P1 重点 | 链表（反转/合并/环） | 能默写 | 8-10 道 |
| P1 重点 | 栈 + 队列 | 能解决括号/单调栈 | 8-10 道 |
| P2 了解 | 基础 DP（爬楼梯/最大子数组） | 能看出是 DP 题 | 5-8 道 |
| ❌ 先不管 | 图论 / 高级 DP / 线段树 / 并查集 | 面试几乎不问 | 0 道 |

**总计：约 70-90 道，覆盖 90% 的面试算法题。**

### 2.2 刷题语言

Java。理由：
- 你本来就是 Java 后端，用 Java 刷题的同时练语法
- 面试时手写 Java 代码，用刷题练出来的手感正好

### 2.3 怎么刷（不是刷了多少道，是会了多少种）

| 错误做法 | 正确做法 |
|----------|----------|
| 每题只做一遍，看懂答案就过 | 按题型刷，同一题型连做 3-5 道 |
| 上来就做 Hard | 先 Easy 理解思路，再 Medium 巩固 |
| 做完就忘，不复习 | 用间隔重复：1天/3天/7天各回顾一次 |

**具体流程：**

```
Step 1：选一个题型（比如双指针）
Step 2：先看 1 道该题型的题解，理解模板
Step 3：连做 3-5 道同类题（不用多，但要自己写出来）
Step 4：写一行费曼笔记 —— "这题型的核心是 ____"
Step 5：3 天后做 1 道同类变种题，检验是否真懂了
```

### 2.4 按周的刷题计划（Month 1）

| 周 | 题型 | 目标 |
|----|------|------|
| Week 1（5/21~5/26）| 数组 + 哈希表 | 连做 10 道，写 5 条题型笔记 |
| Week 2（5/27~6/2）| 双指针 + 滑动窗口 | 连做 10 道，写 5 条笔记 |
| Week 3（6/3~6/9）| 二叉树（遍历+基础） | 默写 4 种遍历模板，做 5 道 |
| Week 4（6/10~6/16）| 二分查找 + 链表 | 各做 5 道，写笔记 |
| Week 5（6/17~6/23）| 栈/队列 + 基础 DP | 各做 5 道 |
| Week 6（6/24~6/30）| 综合复习 + 短板补齐 | 回顾所有题型笔记，补弱项 |

### 2.5 在 AI 时代的定位

Java 后端在 AI 时代不会消失，但会变化。12-15k 的岗位依然大量需要 CRUD + 业务逻辑的开发者。

**你需要掌握的核心能力（简历和面试重点）：**

| 能力 | 重要程度 | 说明 |
|------|----------|------|
| Spring Boot 熟练度 | 🔴 必须 | 项目经验核心，必须能聊透 |
| MySQL + 索引优化 | 🔴 必须 | 后端基本功，面试必问 |
| 基础算法 + 数据结构 | 🔴 必须 | 笔试门槛，过不了就没面试 |
| Redis / 缓存 | 🟡 加分 | 能聊就加分 |
| 微服务 / 分布式基础 | 🟢 加分 | 12-15k 问得不多，了解即可 |
| 会用 AI 写代码 | 🟡 加分 | 面试可以提，但要能解释 AI 写的代码 |

> 12-15k 的面试不要求你架构水平多高。
> 核心考察：**基础扎实 + 能干活 + 沟通顺畅**。

## 三、简历怎么做

### 3.1 简历结构

一份 Java 后端简历，推荐这个结构：

```
一、个人信息（姓名/电话/邮箱/工作经验年数）
二、技术栈（按熟练程度列，别写"精通"）
三、工作经历（最近一份最重要）
四、项目经验（2-3 个，要写结果）
五、教育背景
```

### 3.2 技术栈怎么写

| ✅ 这样写 | ❌ 别这样写 |
|----------|------------|
| 熟练掌握 Java、Spring Boot、MyBatis | 精通 Java、精通 Spring（别写精通） |
| 熟悉 MySQL 索引优化、SQL 调优 | 熟悉 MySQL（太笼统） |
| 熟悉 RESTful API 设计 | 了解 HTTP 协议（太弱） |
| 了解 Redis 基础使用 | 精通 Redis（别夸大） |

### 3.3 项目经验怎么写

项目经验是简历的核心。每条项目写 3 行：

```
第一行：项目名 + 你的角色
第二行：用了什么技术栈 + 做了什么
第三行：取得了什么结果（用数据）
```

示例：

```
某电商后台管理系统 ｜ 核心开发（2024.03 - 2024.08）
- 基于 Spring Boot + MyBatis + MySQL 开发订单模块
- 设计数据库表结构，通过索引优化将查询时间从 800ms 降到 50ms
- 实现订单状态的自动流转，覆盖 10+ 种订单场景
```

> 注意：**第三行要有数据**。"优化性能" → "从 800ms 降到 50ms"，差别巨大。

### 3.4 用 AI 帮你改简历

```
对 AI 说：
"我写了一版 Java 后端简历，帮我 review：
1. 技术栈写得有没有问题
2. 项目描述有没有不够具体的地方
3. 整体是否适合投递 12-15k 的岗位"
```

## 四、面试怎么准备

### 4.1 面试四块内容

| 模块 | 占比 | 准备方法 |
|------|------|----------|
| 算法题 | 30% | 上面刷题计划就行 |
| Java 基础 + Spring Boot | 30% | 八股文 + 自己项目里用过的东西 |
| 数据库（MySQL/索引/SQL）| 20% | 基础原理 + 优化经验 |
| 项目经历 + 行为面 | 20% | 准备好 2-3 个项目的完整叙述 |

### 4.2 Java 后端八股文重点

**P0（必须会）：**
- Java 集合（HashMap 原理、ArrayList vs LinkedList）
- Spring Boot 核心（IoC、AOP、自动配置）
- MySQL 索引原理（B+树、聚簇索引、覆盖索引）
- SQL 调优（explain、慢查询）
- RESTful API 设计规范

**P1（大概率会问）：**
- JVM 内存模型（堆/栈/方法区）
- 垃圾回收（GC 算法、CMS/G1）
- 线程池参数、并发 vs 并行
- Redis 基础数据结构和使用场景

**P2（了解即可，12-15k 不太问）：**
- 微服务（Spring Cloud、服务注册发现）
- 消息队列（RabbitMQ、Kafka）
- 分布式事务

## 六、与学习法则的结合

这套方案里已经内置了学习方法：

| 法则 | 在哪用 | 怎么用 |
|------|--------|--------|
| 费曼 | 每题做完 | 写一行核心思路 |
| 主动回忆 | 复习时 | 不看笔记自己默写代码 |
| 间隔重复 | 题型切换时 | 3 天后做 1 道同类变种题 |
| 刻意练习 | 每天刷题 | 先做 1 道不会的，再做 1 道会的 |
| 80/20 | 选题型 | 只刷高频，冷门先不管 |
| AI 辅助 | 全过程 | 让 AI 出题、批改、模拟面试 |
