## 二、当前只保留什么

### 2.1 刷题语言

Java。理由：
- 你本来就是 Java 后端，用 Java 刷题的同时练语法
- 面试时手写 Java 代码，用刷题练出来的手感正好

### 2.2 怎么刷

| 行为 | 标准 |
|------|------|
| 打开 LeetCode | 视为今天已启动 |
| 有余力做 1 道 | 视为加分，不是义务 |
| 没做题但打开了 | 仍然算完成 |

**具体流程：**

```
Step 1：打开 LeetCode
Step 2：有精力就做 1 道，没有就退出
Step 3：如果做了题，记 1 句笔记
```

### 2.3 当前按周的最小动作

| Week 1 | 打开一次 LeetCode | 建立频率 |
| Week 2 | 打开一次 LeetCode | 不断掉 |
| Week 3 | 打开一次 LeetCode | 维持最低门槛 |
| Week 4 | 打开一次 LeetCode | 仍然算赢 |

### 2.4 在 AI 时代的定位

Java 后端在 AI 时代不会消失，但会变化。12-15k 的岗位依然大量需要 CRUD + 业务逻辑的开发者。

**你需要掌握的核心能力（简历和面试重点）：**

| 能力 | 重要程度 | 说明 |
|------|----------|------|
| Spring Boot 熟练度 | 🔴 必须 | 项目经验核心，必须能聊透 |
| MySQL + 索引优化 | 🔴 必须 | 后端基本功，面试必问 |
| 基础算法 + 数据结构 | 🔴 必须 | 笔试门槛，过不了就没面试 |
| Redis / 缓存 | 🟡 加分 | 能聊就加分 |
| 微服务 / 分布式基础 | 🟢 加分 | 12-15k 问得不多，了解即可 |
| 会用 AI 写代码 | 🟡 加分 | 面试可以提，但要能解释 AI 写的代码 |

> 12-15k 的面试不要求你现在就把所有题都刷完。
> 先活下来，再慢慢加。
