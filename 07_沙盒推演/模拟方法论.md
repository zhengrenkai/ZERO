# 模拟方法论

> 零系统的推演不是凭感觉猜，是用已知数据+逻辑推演。这套框架确保每次推演都是可复现、可验证、可迭代的。

---

## 一、核心框架

每次推演遵循 **4 步流程**：

```
① 采集输入 → ② 分层推演 → ③ 识别风险 → ④ 输出行动
```

### ① 采集输入

推演前必须读取以下数据，不能凭记忆：

| 数据源 | 采集什么 | 为什么重要 |
|--------|----------|-----------|
| 01_人生核心档案/行为模式.json | 已知失败模式、风险模式、动力模式 | 决定推演的假设条件 |
| 01_人生核心档案/当前状态.json | 当前 5 维状态（精力/动量/纪律/疲劳/压力） | 决定推演的起点 |
| 01_人生核心档案/人格与方向.json | 性格特征、人生方向 | 判断面对压力时的行为倾向 |
| 01_人生核心档案/人格与方向.json | 性格特征、人生方向、工作约束、居住环境 | 决定推演的外部约束 |
| 02_目标系统/（当前月+周） | 当前的实际目标和计划 | 推演的对象 |
| 相关模块（03_交易/02_目标） | 具体的执行方案 | 推演的细节输入 |

### ② 分层推演

从三个层级依次推演，不跳级：

```
第 1 层：时间预算层
  每天有多少可用时间？每条线各占多少？
  冲突点在哪里？(如加班、周末)
  输出：时间表 + 冲突矩阵

第 2 层：状态变化层
  基于起点状态 + 每日消耗，推演 4-12 周的精力/疲劳/动量变化
  考虑已知模式（如 Week 4 崩溃、熬夜断训）
  输出：每周状态曲线

第 3 层：风险识别层
  在时间预算和状态变化的交叉点找风险
  输出：风险清单（等级+概率+影响+触发时间）
```

### ③ 识别风险

每个风险必须有 4 个要素：

```
风险 = 什么会出问题
概率 = 发生的可能性（高/中/低）
影响 = 如果发生了，后果多严重
触发时间 = 大约在推演的第几周
```

风险分级：
- 🔴 致命 — 系统可能完全中断
- 🟡 严重 — 某条支柱停摆
- 🟢 中等 — 执行率下降但可恢复

### ④ 输出行动

每个识别的风险必须有对应的行动：

```
发现风险 → 预防措施（做什么来降低概率）
         → 应急预案（发生了怎么办）
         → 最早干预节点（什么时候做第一次检查）
```

## 二、推演时间窗

| 推演类型 | 时间跨度 | 适用场景 | 预计耗时 |
|----------|----------|----------|----------|
| 完整推演 | 3 个月 | 系统重大变更后 | 30-45min |
| 专项推演 | 1-2 个月 | 某条线出现瓶颈 | 15-20min |
| 快速推演 | 2-4 周 | 每周复盘前，检查下周风险 | 5-10min |

## 三、推演假设管理

每次推演必须记录假设条件。假设分三类：

| 假设类型 | 含义 | 例子 |
|----------|------|------|
| 硬假设 | 不变的约束 | "工作日 9-18 上班"、"加班频率偶尔" |
| 软假设 | 大概率成立但不绝对 | "每周与 AI 对话至少 5 次" |
| 行为假设 | 基于已知模式的行为预测 | "Week 3-4 会出现疲劳累积" |

**推演后检视**：记录到 `推演记录/` 的推演报告中，3 个月后回看哪些假设对了、哪些错了。

## 四、多维推演

### 4.1 时间推演

计算每天的"可用时间预算"，识别时间冲突点：

```
方法：
1. 列出一天的固定时间块（工作/通勤/吃饭/睡觉）
2. 插入计划任务（训练/交易学习/刷题）
3. 计算剩余自由时间
4. 标记冲突点（如午休刷题与吃饭冲突）
```

### 4.2 状态推演

基于起点状态，模拟每周的状态变化：

```
方法：
1. 从当前状态 5 维数值开始（精力/动量/纪律/疲劳/压力）
2. 每周一个调整因子（+1 / 0 / -1 / -2）
3. 考虑已知模式的触发条件（如疲劳 ≥ 8 时触发完美主义）
4. 输出 4-12 周的状态曲线
```

调整因子参考：

| 因素 | 对精力的影响 | 对疲劳的影响 | 对动量的影响 |
|------|-------------|-------------|-------------|
| 连续执行 1 周 | +1 | -1 | +1 |
| 周末有恢复 | +1 | -1 | 0 |
| 周末无恢复（8h 学习） | 0 | +1 | +1→0 |
| 中断 1-2 天 | -1 | 0 | -1 |
| 交易学习无反馈 | 0 | +1 | -1 |

### 4.3 风险推演

将时间冲突和状态变化交叉分析：

```
方法：
1. 在时间预算表中标记"高风险时段"（如每晚 20:10 的交易学习）
2. 在状态曲线上标记"临界点"（如疲劳 ≥ 8 的周）
3. 找出交叉点（如风险时段 + 临界点 = 高风险场景）
4. 推演最可能的崩溃路径
```

## 五、推演偏差校正

推演不是预言。偏差是正常的，关键是从偏差中学习。

| 偏差类型 | 常见原因 | 校正方法 |
|----------|----------|----------|
| 过于乐观 | 低估了执行的阻力 | 下次推演使用更保守的执行率 |
| 过于悲观 | 忽略了正反馈的作用 | 加入"习惯形成"的正向因子 |
| 忽略随机性 | 没有考虑加班/生病等随机事件 | 加入"随机中断"因子（每月 2-3 天） |

## 六、推演迭代

每次新推演与上一次推演对比：

| 对比维度 | 问题 | 目的 |
|----------|------|------|
| 假设变化 | 这次的假设和上次有哪些不同？ | 验证假设调整是否合理 |
| 风险变化 | 上次识别的风险这次是否仍然存在？ | 评估防御措施的效果 |
| 精度变化 | 上次的预测和实际差距多大？ | 改进推演方法 |
| 新发现 | 这次发现了上次没看到什么？ | 拓展推演视野 |

## 七、推演会议流程

当 AI 运行推演时，按此流程执行：

```
1. 读取所有输入数据（5min）
   → 行为模式.json、当前状态.json、人格与方向.json、环境信息.json
   → 当前月度目标 + 周计划
   → 相关模块的执行方案

2. 时间预算分析（5min）
   → 计算工作日/周末的可用时间
   → 标记冲突点

3. 状态变化推演（10min）
   → 从起点状态开始，逐周推演
   → 标记关键转折点

4. 风险识别（10min）
   → 列出所有可能的风险
   → 分级、评估概率和影响
   → 确定最早干预节点

5. 输出行动（5min）
   → 写入推演报告
   → 更新优化建议池
   → 更新迭代路线图
```
