# 零的愿景：你要变成什么样

> 一个系统如果不知道要去哪里，那它永远不会到位。
> 这份文档就是"零"自己的北极星。

---

## 一句话定义

**零是一个人一生都在用的、以文件为基础、以 AI 为引擎的个人操作系统。**

它不是 App，不是网页，不是数据库——它是一套方法 + 一套文件 + 一个 AI 角色，三者合一。

---

## 四个阶段进化路径

### Phase 0：现在（2026-05）

你现在就在这里。

```
定义：个人备忘录系统
核心：用户手动维护 JSON + AI 在对话中查看
范围：身体训练（1 个子系统）
能力：AI 能读取状态、给出策略、记录日志
局限：
  - 完全依赖 AI 对话
  - 所有文件手动维护
  - 只有身体系统在跑
  - 没有系统自身的迭代机制
```

### Phase 1：完整系统（1-3 个月）

```
定义：个人目标操作系统
核心：文件系统 + AI 对话 + 辅助脚本
范围：身体系统 + 目标系统 + 复盘系统 + 交易系统
能力：
  - 所有子系统都有最小可用模板
  - 复盘标准化：每周围绕数据和模式复盘
  - 交易系统冷启动：有最小模板和风控规则
  - 仪表盘自动生成（已有）
  - JSON Schema 校验（已有）
  - 每个模块不再有 .gitkeep
新增：09_系统建设/ 自我迭代机制
```

### Phase 2：个人仪表盘（3-6 个月）

```
定义：带可视化反馈的个人操作系统
核心：文件系统 + AI 对话 + 状态可视化
能力：
  - 状态趋势图：精力/动量/纪律的时间线
  - 执行率统计仪表盘
  - 每周自动生成周报
  - 多设备同步方案（Git 已经是，但需要更友好）
  - 移动端可查看（通过 GitHub 或其他方式）
  - 交易系统跑起来
新增：可视化层、周报自动生成
```

### Phase 3：生活副驾驶（6-12 个月）

```
定义：AI 主动驱动的个人成长系统
核心：文件系统 + AI 对话 + 模式识别 + 主动建议
能力：
  - AI 自动检测行为模式变化
  - 跨子系统分析：训练状态是否影响工作状态
  - 风险预警自动化
  - 目标冲突检测（如：瘦身和增肌冲突）
  - 长期趋势报告
  - 可以回答"我这三个月进步了什么"
新增：模式识别引擎、跨域分析、长期报告
```

### Phase 4：终极形态（12 个月+）

```
定义：一个人用一生的操作系统
核心：文件系统 + AI 对话 + 知识图谱 + 时间线
能力：
  - 一生的时间线：从今天回溯到系统创建那天
  - 知识沉淀：所有踩过的坑、悟到的道理、形成的原则
  - 决策辅助：基于历史数据给出最优选择
  - 跨生命周期管理：职业、健康、财务、关系、认知
  - 可以回答"你的一生告诉了你什么"
新增：知识图谱、终身时间线、决策引擎
```

## 零的底层原则

无论进化到哪个阶段，这些不变：

| 原则 | 含义 | 违背后果 |
|------|------|----------|
| 文件优先 | 所有数据最终落地为文件，不依赖特定平台 | 锁死在某个 App 里 |
| 对话为界面 | 人和系统的交互通过 AI 对话完成 | 系统变成冷冰冰的数据库 |
| 极简启动 | 每个子系统从最小可用开始，永远不加过量 | 第 4 周崩溃 |
| 渐进迭代 | 不从 Phase 1 跳到 Phase 3，一步步走 | 系统复杂度失控 |
| 持久化记忆 | 每次对话结束时更新核心文件 | AI 每次都是失忆状态 |
| 不离不弃 | 中断了可以重启，没有"失败"只有"数据" | 完美主义放弃 |

## 现在你在哪里

```
您现在在这里 → Phase 0
                      |
               Phase 1 → 下一步
                      |
               Phase 2 → 如果系统稳定运行 3 个月
                      |
               Phase 3 → 如果跑通 2 个以上子系统
                      |
               Phase 4 → 长期目标
```

> 你的直接任务是：**把 Phase 0 推到 Phase 1**。
> 也就是：把剩下的 .gitkeep 全部填上最小模板，让每个模块至少"能用"。
