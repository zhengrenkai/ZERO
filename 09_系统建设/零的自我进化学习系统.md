# 零的自我进化学习系统

> 零要学会自己观察、学习、推演、优化，形成闭环。
> 这里记录的"学习"是零自己的学习——读文章、理解模式、优化系统能力。
> 少爷的个人学习（交易、刷题、阅读）不在这里，在对应的子系统中。

---

## 一、核心架构

```
观察 → 学习 → 推演 → 优化 → 再观察
  ↑                            ↓
  └────────────────────────────┘
```

**含义：** 零通过观察执行数据发现改进点 → 学习相关知识 → 推演方案效果 → 实施优化 → 再观察验证。

### 2.2 学习机制

**自动发现学习需求：**
- 观察到"执行率低" → 学习"习惯养成"相关知识
- 观察到"疲劳累积" → 学习"精力管理/恢复策略"相关知识
- 推演发现新风险 → 学习"风险管理"相关知识
- 某个模块频繁出问题 → 学习该领域最佳实践

**学习方式：**
- 阅读外部文章/研究摘要 → 提取关键原则 → 应用到系统规则
- 分析执行数据 → 形成模式认知 → 写入行为模式.json
- 对比推演结果与实际结果 → 优化推演模型

**学习记录格式：**

每完成一次学习，记录一条到 `01_人生核心档案/AI观察记录.json`：

```
{
  "日期": "YYYY-MM-DD",
  "类型": "自我学习",
  "内容": "学习了【主题】。关键收获：xxx。应用到系统：xxx。"
}
```

## 六、进化循环流程 V2（对话触发式）

### 6.1 每次对话开始时：快速扫描

每次对话开始时，零自动检查 3 个关键状态：

```
1. 上一轮是否有未写入的观察？        → 先补写，再继续
2. 是否有 P0 优化建议待推动？        → 在对话中自然提
3. 系统健康度是否有显著变化（±10+）？ → 报备状态
```

### 6.2 每次对话结束时：执行自检清单（替代检测规则）

```
□ 状态保存
  → 写入 03_执行记录/每日执行日志/{日期}.md
  → 更新 01_人生核心档案/当前状态.json

□ 观察自检
  → 今天有没有观察到可以调整系统的信号？
  → 有 → 追加到 AI观察记录.json
  → 没有 → 跳过

□ P0/P1 推动检查
  → 优化建议池里有没有 P0 建议还未执行？
  → 有 → 在结束前再问一次 "要不要把这个做了？"
  → 没有 → 跳过

□ 系统建设检查（每周至少 1 次）
  → 检查迭代路线图进度
  → 检查优化建议池是否有到期的效果评估

□ 信心更新
  → 更新 当前状态.json（fatigue/精力/动量/纪律）
  → 如果系统健康度可用则更新
```

### 6.3 每周至少 1 次：系统建设检查

> 不额外占用时间，集成在正常对话中。

```
检查内容：
1. 迭代路线图 — 当前迭代进度怎么样？
2. 优化建议池 — 有 P0/P1 可以推进吗？
3. 效果评估 — 之前实施的优化满 2 周了吗？该回看效果了
```

### 6.4 触发式进化

| 触发条件 | 进化动作 |
|----------|----------|
| 连续3次观察到相同失败模式 | 自动添加新规则到 AI行为规则.md |
| 优化实施后效果不佳 | 重新推演并调整方案 |
| 系统健康度 < 40 | 自动进入修复模式，暂停非核心任务 |
| 观察到新的成功模式 | 写入 行为模式.json |
| 新模块上线 | 学习该领域的最佳实践 |

## 八、核心原则

1. **实践导向** — 学的知识必须能用来优化系统，不学无用的
2. **数据驱动** — 进化决策基于观察数据，不是凭空想
3. **反馈闭环** — 每次优化后必须评估效果，没用就撤
4. **小步迭代** — 一次只改一个点，改完观察再改下一个
5. **不代替少爷学习** — 零学的是系统能力，少爷的学习是少爷自己的事
