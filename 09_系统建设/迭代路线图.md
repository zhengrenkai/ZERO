## 迭代节奏

### 每周一个"系统建设日"

建议每周找 30 分钟做一次系统建设，完成：

```
1. 检视当前迭代进度（路线图上的任务完成了几项）
2. 选择下一个要做的任务
3. 创建模板或完善模块
4. 更新本文件
```

### 迭代规则

1. **一次只推一件事**。不要同时做复盘模板 + 交易系统 + 模板开发
2. **先模板，再内容**。先让结构到位，内容等实际使用时自然填充
3. **检验标准是"是否被用过一次"**。模板创建了但从未使用 = 不存在
4. **AI 在每次对话中检查一次迭代进度**。如果用户有进展，记录和鼓励
