# 学习法则

> 不是让你学更多，是让你学过的每样东西都变成自己的。
> 站在巨人的肩膀上——这里的方法论来自费曼、比约克、埃里克森等，但只写有实操价值的。

---

## 法则一：费曼学习法

### 一句话
**学不懂的东西，试着讲给一个完全不懂的人听。讲不通就是没真懂。**

### 怎么做

```
Step 1：学完一个概念 → 关掉资料
Step 2：用自己的话写/讲一遍（假设对方完全不懂）
Step 3：讲不下去的地方 → 回去再看
Step 4：简化。去掉术语，用大白话。
```

### 在刷题中的应用

```
做完一道题，不急着交卷。
写下这行字：
"这道题的核心思路是 ____"
→ 如果写不出来，说明你还没真懂

示例：
"Two Sum 的核心思路是：遍历数组时，
把每个数需要的差值存到 map 里，
后面的数如果刚好等于这个差值，就配对成功。"
```

### 在交易中的应用

```
看完流动性这一篇课，打开一个空白文档，
写下这行字：
"流动性用大白话说就是 ____"
→ 如果卡住了，回去重看那一段

示例：
"流动性就是价格会去的地方。
因为那里有很多止损单，价格会扫过去吃掉这些流动性。
你可以把前高和前低当成流动性区域。"
```

## 法则二：主动回忆

### 一句话
**不看书，纯靠脑子想。想不出来的就是没记住的。**

### 为什么有效
看一遍觉得自己懂了 → 是错觉（流利度错觉）。
合上书试着回想 → 才是真的检验。

### 在刷题中的应用

| 学习方法 | 效果 |
|----------|------|
| 看答案 → 觉得懂了 → 下一题 | ❌ 假懂 |
| 看题目 → 自己写 → 写不出再看答案 → 重新写一遍 | ✅ 真会 |

**操作：**
- 打开一道题 → 先自己想 5 分钟
- 想不出来 → 看答案 → **关掉答案** → 自己默写一遍
- 写错的地方 → 就是你还没真会的地方

### 在交易中的应用

```
看完流动性这一篇课 → 关掉视频 → 打开行情图
给自己出题：
"这张图上，流动性区域在哪里？标出来。"
→ 标对了 → 你真懂了
→ 标不出来 → 回去重看
```

## 法则三：间隔重复

### 一句话
**在快要忘记的时候复习，效果是死记硬背的 10 倍。**

### 复习节奏

```
学完当天   → 写笔记（用自己的话）
第 2 天    → 花 5 分钟回顾标题
第 4 天    → 做一次主动回忆（不看笔记回想）
第 7 天    → 做一道相关练习题
第 14 天   → 做一次综合应用
```

### 在刷题中的应用

```
周一学完「二分查找」→ 做 3 道题
周二花 5 分钟：回想二分查找模板
周四不看笔记：默写二分查找代码
周日再做 1 道二分查找变种题
两周后：如果还能写出来 → 真会了
```

### 在交易中的应用

```
周一学完流动性 → 在图上标 3 个流动性区域
周二复盘前：先自己回想流动性的定义
周四：不看笔记，再找 3 个流动性区域（新图表）
周日：综合运用流动性和市场结构分析行情
```

## 法则四：刻意练习

### 一句话
**不要反复做你已经会的事，做你不会的。**

### 在刷题中的应用

```
大多数人犯的错：
  会做数组题 → 一直做数组题（舒适区）
  不会动态规划 → 逃避动态规划（恐惧区）

正确做法：
  每天先花 20 分钟做 1-2 道你不太擅长的题型
  做完再花 10 分钟做你已经会的（当热身）
```

### 在交易中的应用

```
大多数人犯的错：
  只会找流动性区域 → 一直只找流动性区域
  不会识别市场结构 → 直接跳过

正确做法：
  今天专门练识别市场结构转换（MSS）
  哪怕找错了也比不做强
  找到 5 个例子才换下一个概念
```

## 法则五：80/20 法则

### 一句话
**20% 的核心内容，决定了 80% 的结果。先抓住那 20%。**

### 在刷题中的应用

面试最常考的题型（按频率排列）：

```
高频 80%：
  数组/字符串 → 双指针/滑动窗口 → 哈希表
  → 二叉树遍历 → 二分查找 → 链表基础
冷门 20%（先不管）：
  图论 → 高级DP → 线段树 → 并查集
```

**策略：先只刷高频 80%，高频掌握到能秒 Medium 了，再碰冷门。**

### 在交易中的应用

交易之境 7 篇里，最核心的 20%：

```
最核心（先掌握）：
  ① 流动性原理 — 读懂价格行为的基础
  ② 市场结构 — 判断趋势和反转
  ③ POI 区域 — 找到入场点
后 80%（然后再学）：
  ④ 价格传递 — 精细化入场
  ⑤ 背驰判断 — 辅助过滤器
  ⑥ 多周期 — 提高准确率
  ⑦ 交易系统 — 整合
```

**策略：先把前三篇练到闭眼能讲，再看后面的。**

## 法则六：AI 辅助学习法

### 一句话
**让 AI 当你的私人老师，而不是替你写答案。**

### 用法清单

| 场景 | 你该怎么说 | AI 该做什么 |
|------|-----------|------------|
| 概念卡住了 | "用大白话解释一下什么是滑动窗口" | 用最简单的语言解释 + 给 1 个生活例子 |
| 需要出题 | "给我出一道二分查找的变种题，Medium 难度" | 出题 + 答案 + 思路讲解 |
| 批改答案 | "这道题我的解法是这样… 帮我 review 一下" | 指出优化空间 + 复杂度分析 |
| 模拟面试 | "模拟一场 Java 后端的电话面试" | 问技术问题 + 反馈回答质量 |
| 交易复盘 | "这张图我标了流动性区域，你看看对不对" | 指出错误 + 解释为什么错 |
| 概念检验 | "我用大白话讲一下订单块，你帮我判断对不对" | 指出理解偏差 + 补充遗漏 |

### 核心原则
**永远不要直接问答案。**
先自己思考 → 再让 AI 帮你检验 → 而不是让 AI 替你想。

## 这些法则怎么组合用

### 刷题时的 30 分钟流程

```
前 5 分钟：主动回忆（回想昨天做的题型模板）
中间 20 分钟：刻意练习（做 1 道你不擅长的 + 1 道你擅长的）
最后 5 分钟：费曼输出（写一行核心思路）
```

### 交易学习时的 1 小时流程

```
前 15 分钟：间隔回顾（回想前几天学的概念，在图上找）
中间 30 分钟：刻意练习（只练今天瞄准的一个概念）
最后 15 分钟：费曼输出（给 AI 讲一遍你今天的发现）
```
