# 对话结束检测规则

> 定义零系统如何判断对话何时结束，并触发结束流程。

---

## 一、结束信号类型

### 1. 明确结束信号（优先级最高）

当用户发送以下内容时，立即判定为对话结束：

| 关键词/短语 | 示例 |
|-------------|------|
| 结束对话 | "今天就这样，结束对话" |
| 退出 | "退出" |
| 再见 | "再见" |
| 拜拜 | "拜拜" |
| 今天就到这 | "今天就到这吧" |
| 先这样 | "先这样，明天再说" |
| 下次聊 | "下次再聊" |

### 2. 隐含结束信号

当用户发送以下内容时，判定为对话结束：

| 场景 | 示例 |
|------|------|
| 确认完成 | "好的，我知道了"、"明白了"、"谢谢" |
| 执行计划确认 | "今天就这样安排"、"按你说的做" |
| 已完成汇报 | "今天的执行情况就是这样" |

### 3. 超时结束（兜底机制）

当满足以下条件时，自动判定为对话结束：

- 最后一次用户输入后 **5分钟** 无响应
- 最后一次AI回复后 **3分钟** 无响应

---

## 二、结束检测流程

```
用户输入 → 检查是否有明确结束信号
              ↓
         有 → 立即触发结束流程
              ↓
         无 → 检查是否有隐含结束信号
              ↓
         有 → 延迟10秒后触发结束流程（给用户补充的机会）
              ↓
         无 → 继续监听（触发超时计时）
              ↓
         超时 → 触发结束流程
```

---

## 三、结束流程执行内容

触发对话结束后，按以下顺序执行：

1. **状态保存**
   - 写入 `03_执行记录/每日执行日志/{日期}.md`
   - 更新 `01_人生核心档案/当前状态.json`

2. **观察自检**
   - 思考："今天有没有观察到可以调整系统的信号？"
   - 有 → 追加一条到 `AI观察记录.json`（`scope: instant` 或 `scope: trend`）
   - 没有 → 跳过

3. **系统建设检查**（每周末额外执行）
   - 运行 `src/system_build_check.js`
   - 更新 `09_系统建设/自动化检查报告.md`

4. **推送更新**
   - `git status` 确认改动范围
   - `git commit + push`（按 conventional commit 格式）

---

## 四、结束提示

对话结束时，输出统一的结束提示：

```
少爷，今天的对话就到这里。

📝 已保存今日执行记录
📊 已更新状态
🔍 已完成观察自检

下次见！
```

## 五、代码实现（待添加到 src/）

未来可创建 `src/end_detection.js`，实现以下功能：

```javascript
// 检测是否为结束信号
function isEndSignal(message) {
  const endKeywords = ['结束', '退出', '再见', '拜拜', '今天就到这', '先这样', '下次聊'];
  return endKeywords.some(keyword => message.includes(keyword));
}

// 检测是否为隐含结束信号
function isImplicitEnd(message) {
  const implicitKeywords = ['知道了', '明白了', '谢谢', '按你说的做', '今天就这样安排'];
  return implicitKeywords.some(keyword => message.includes(keyword));
}

// 执行结束流程
function executeEndProcess() {
  // 1. 保存执行记录
  saveDailyLog();
  
  // 2. 更新状态
  updateCurrentState();
  
  // 3. 观察自检
  checkObservations();
  
  // 4. 推送更新
  pushUpdates();
  
  // 5. 输出结束提示
  outputEndMessage();
}
```

## 六、特殊情况处理

| 情况 | 处理方式 |
|------|----------|
| 用户中途离开后又返回 | 重新开始对话，保留之前的状态 |
| 对话被意外中断 | 下次对话时自动检测未完成的结束流程并补全 |
| 周末/月末流程未完成 | 下次对话时自动检测并补全 |
