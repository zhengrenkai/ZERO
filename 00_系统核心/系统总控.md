## 二、系统分层

### 2.1 三层结构

| 层级 | 目录 | 用途 | 规则 |
|---|---|---|---|
| 当前执行层 | `05_仪表盘/` + `02_目标系统/每日任务/` | 只放今天/本周真正要看的内容 | 日常启动默认只读这一层与必要状态 |
| 规划控制层 | `00_系统核心/` + `01_人生核心档案/` + `02_目标系统/` | 放规则、目标、阶段、唯一数据源 | 仅在维护模式或必要更新链中扩展读取 |

## 三、数据源与展示层

### 3.1 唯一来源规则

| 信息类型 | 唯一来源 | 展示文件 |
|---|---|---|
| 5维状态 | `01_人生核心档案/当前状态.json` | `05_仪表盘/当前人生状态.md` |
| 今日执行安排 | `02_目标系统/每日任务/YYYY-MM-DD.md` | `05_仪表盘/今日任务.md` |
| 今日最小动作 | 从当日每日任务派生 | `05_仪表盘/独立执行卡.md` |
| 阶段/里程碑 | `02_目标系统/总体推进计划.md` | 启动简报引用 |
| 历史执行记录 | `02_目标系统/归档/` 与 `08_复盘系统/` | 仅维护模式读取 |

### 3.2 镜像压缩规则

1. 日常信息只允许存在 **1 个数据源 + 1 个展示层**。
2. 禁止同一状态散落在 3 个以上文件。
3. 历史记录不反向影响当前执行层，除非进入维护模式。
4. 低频文件不得占用高频入口。

## 四、启动与维护工作流

### 4.1 日常启动（默认）

适用：`启动`、`今日任务`、简短汇报、状态判断。

```text
第1步：读取 00_系统核心/系统总控.md
第2步：读取 01_人生核心档案/当前状态.json
第3步：读取 05_仪表盘/当前人生状态.md
第4步：读取 05_仪表盘/今日任务.md + 独立执行卡.md
第5步：按需引用 02_目标系统/总体推进计划.md 中的当前阶段信息
第5.5步：默认遵守 `总体推进计划.md` 中的统一推进规则（基线 / 进阶 / 降级 / 产出 / 周预算）
第6步：输出简报：档位 + 昨夜回顾 + 今日唯一动作 + 晚间底线
```

### 4.2 系统维护（按需）

适用：`进入维护`、`维护系统`、`检查系统`、`检查一致性`、改规则、改目标、归档历史。

```text
第1步：执行日常启动
第2步：读取规划控制层：01_人生核心档案/ + 02_目标系统/ + 必要规则
第3步：按需读取 03_交易系统/、04_现金流稳定系统/、07_沙盒推演/
第4步：按需读取历史归档层：02_目标系统/归档/ + 08_复盘系统/
第5步：输出问题清单 + 修改建议 + 一致性检查结果
第6步：改动后执行结构/引用/同步检查
```

### 4.3 维护模式新增职责

- 扫描目录职责是否混乱
- 扫描当前执行区是否混入历史内容
- 扫描引用是否断链
- 扫描镜像是否过多
- 扫描是否有低频文件占据高频入口
- 检查当前建议是否符合 `总体推进计划.md` 的统一推进规则

## 七、一致性检查

### 7.1 每次维护后必须检查

| 检查项 | 方法 |
|---|---|
| 系统总控中的目录树 = 磁盘实际目录 | `Get-ChildItem` 对比 |
| `05_仪表盘/` 只保留 3 个当前执行文件 | 目录检查 |
| `02_目标系统/每日任务/` 是否只保留活跃窗口 | 目录检查 |
| 历史每日任务是否进入归档区 | 检查 `02_目标系统/归档/每日任务/` |
| 5维状态唯一来源是否明确 | 核对 `当前状态.json` → `当前人生状态.md` |
| 今日任务唯一来源是否明确 | 核对当日每日任务 → `今日任务.md` |
| 模板是否占用高频入口 | 检查模板是否留在 `00_系统核心/模板/` |
| 所有路径引用是否真实存在 | `rg` + 目录检查 |

### 7.2 发现问题时的动作

- 目录混乱：先重分层，再修引用
- 数据源重复：保留唯一来源，删除多余镜像
- 当前执行层混入历史：立即归档
- 模板混入仪表盘：立即迁回模板层

## 八、执行档位

| 档位 | 条件 | 系统动作 |
|---|---|---|
| 保命档 | 疲劳 8-10 | 只保睡眠底线 + 身体最低动作，交易/现金流稳定线暂停 |
| 恢复档 | 疲劳 6-7 | 身体最低版 + 交易15min，现金流稳定线只保打开 |
| 正常档 | 疲劳 4-5 | 三线可执行，但一次只推一件事 |
| 增强档 | 疲劳 ≤ 3 且连续稳定 | 只小幅加量，不同时加多条线 |

## 九、核心原则

1. **我是整个系统，不是部分系统。**
2. **系统维护优先于系统变重。**
3. **日常入口必须省力。**
4. **当前执行层只放当前。**
5. **历史要保留，但必须归档。**
6. **禁止空谈，建议必须可执行。**
7. **一次只推一件事。**
8. **状态差先保命，状态稳再推进。**
9. **所有涉及时间的判断必须先用 `Get-Date`。**
