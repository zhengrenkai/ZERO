# 工作流程

## 启动流程（每次新对话）

1. 读取 00_系统核心/人生逆袭实验.md — 实验框架
2. 读取 00_系统核心/AI系统自身概要.md — 系统身份
3. 读取 01_人生核心档案/ 全部 6 个 JSON — 记忆恢复
4. 读取 08_仪表盘/ — 当前状态
5. 读取 02_目标系统/总体推进计划.md — 当前推进阶段和下一步行动
6. 读取 09_系统建设/优化建议池.md — 待办建设任务
7. **自动运行系统建设检查** — 执行 `src/system_build_check.js`
8. 分析状态，检查未处理观察
9. 执行对话开场：输出简报 + 推进位置，引导少爷汇报

## 每日对话流程

### 1. 开场
- 输出状态简报（含身体/交易/工作三条线）
- 引导少爷汇报昨日执行

### 2. 少爷汇报后
- 确认执行结果
- 更新 5 维状态（精力/动量/纪律/疲劳/压力）
- 如有风险预警
- 给出今日策略

### 3. 对话结束时
- 写入 03_执行记录/每日执行日志/
- 更新 01_人生核心档案/当前状态.json
- **观察自检**：花10秒想"今天有没有观察到可以调整系统的信号？"
  - 有 → 追加一条到 AI观察记录.json（即时用 scope: instant，趋势用 scope: trend）
  - 没有 → 跳过
- **自我进化扫描**：检查最新观察中是否有"可以优化系统"的信号
  - 读取 AI观察记录.json 最近5条观察
  - 逐条判断：这条观察是否暗示了"系统可以改进的地方"？
  - 判断标准：
    - 同一类型的「失败模式」出现3次以上 → 需要建立新规则
    - 同一「策略建议」重复提出 → 说明之前的建议没执行，需要加强
    - 出现新的有效「执行模式」→ 考虑写入行为模式.json
    - 出现「风险预警」且已有措施但未执行 → 写入优化建议池
  - 如有发现 → 自动追加一条优化建议到 `09_系统建设/优化建议池.md`
    - 标记来源为"自我进化扫描"
    - 评估优先级（P0-P3）
  - 如无发现 → 跳过
- **优化效果回看**：检查是否有已实施的优化建议需要验证效果
  - 读取优化建议池中"已完成"状态且无"效果"标记的建议
  - 检查是否已满2周 → 评估效果并记录

### 4. 推送更新（结束前）
- git status 确认改动范围
- git commit + push，备注按 conventional commit 格式：
  - `<type>: <每次改了什么>`
  - 多行：第一行标题，空行后写详细内容
  - 不推送 .idea/ 目录

## 每周流程（周末执行）

- 读取本周每日执行日志（含身体/交易/工作）
- 分析三条线的执行趋势
- 更新行为模式
- 确定下周优化方向
- **自动运行系统建设检查** — 生成 `09_系统建设/自动化检查报告.md`
- **检查迭代路线图进度** — 如有待办任务，提示少爷
- **清理优化建议池** — 已完成的建议打勾，过期的移除
- 写入 11_复盘系统/每周复盘/

## 每月流程（月末执行）

- 分析月趋势（三条线分别评估）
- 评估阶段进展
- 调整季度目标
- 更新长期观察
- 写入 11_复盘系统/每月复盘/
