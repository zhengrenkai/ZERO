# 零 · 用户手册

> 这份手册只回答一个问题：少爷平时怎么用这个系统最省力。
> 原则：少决策、少翻文件、少解释、少维护。

---

## 一、日常使用流程

适用场景：每天打开对话、想知道今天做什么、晚上回来汇报。

### 1. 启动

少爷只需要说：

```
启动
```

零会读取：

```
00_系统核心/系统总控.md
01_人生核心档案/当前状态.json
05_仪表盘/今日任务.md
05_仪表盘/独立执行卡.md
02_目标系统/总体推进计划.md
```

然后直接输出：

- 当前档位：保命档 / 恢复档 / 正常档 / 增强档
- 今日唯一核心动作
- 晚间底线
- 需要砍掉的任务

### 2. 执行

日常只看两个文件：

| 文件 | 用途 |
|------|------|
| `05_仪表盘/今日任务.md` | 看今天完整安排 |
| `05_仪表盘/独立执行卡.md` | 累的时候只看最小动作 |

如果不想看文件，就只听零的简报。

### 3. 汇报

晚上做完后，少爷只需要简短汇报：

```
训练完成，手机放客厅了
```

或：

```
今天没做，太累了
```

不用解释太多。零负责记录、判断、调整。

## 二、执行档位

系统每天先判断档位，再决定做多少。

| 档位 | 条件 | 今日策略 |
|------|------|----------|
| 保命档 | 疲劳 8-10 | 只保睡眠底线 + 身体最低动作，交易/工作暂停 |
| 恢复档 | 疲劳 6-7 | 身体最低版 + 交易 15min，工作只打开 |
| 正常档 | 疲劳 4-5 | 三线都可执行，但一次只推一件事 |
| 增强档 | 疲劳 ≤3 且连续稳定 | 小幅加量，不同时加多条线 |

当前默认逻辑：

```
先不崩 → 再恢复 → 再连续 → 再加量
```

## 三、周末使用流程

适用场景：周六、周日、去对象家、环境变化。

### 周末只保三件事

1. 起床后先吃早餐和补剂。
2. 午饭后可以躺，但交易只能降形式，不降内容。
3. 晚上仍然执行手机隔离和睡眠底线。

### 周日额外规则

周日晚上提前收口：

```
22:00 后不碰刺激内容
22:30 手机放客厅
23:00 尽量躺下
```

如果周日也是月底，零会提醒月度复盘。

## 四、维护使用流程

适用场景：改规则、改目标、复盘、检查系统一致性。

少爷可以直接说：

```
进入维护
```

或：

```
检查系统
```

零会额外读取相关模块：

| 场景 | 读取 |
|------|------|
| 改目标 | `02_目标系统/` |
| 改交易 | `03_交易系统/` |
| 改工作 | `04_工作系统/` |
| 做复盘 | `08_复盘系统/` |
| 改系统规则 | `00_系统核心/系统总控.md` |

维护结束后必须做一致性检查：

1. 当前状态是否同步到仪表盘。
2. 今日任务是否和每日任务一致。
3. 周计划日期是否真实。
4. 总体推进计划是否仍然对齐现实。
5. 需要时提交并推送。

## 五、少爷常用指令

| 少爷说 | 零应该做 |
|--------|----------|
| `启动` | 日常启动，输出今日档位和唯一动作 |
| `今日任务` | 只输出今天要做什么 |
| `我累` | 自动切保命档或恢复档 |
| `训练完成` | 打勾并更新状态 |
| `手机放客厅了` | 记录晚间底线完成 |
| `进入维护` | 读取相关系统，开始改规则 |
| `复盘` | 进入周/月复盘流程 |
| `结束对话` | 执行结束更新链 |

---

## 六、使用原则

- 日常不要翻系统，先说 `启动`。
- 累的时候只看 `独立执行卡.md`。
- 状态差时，不补任务，不加任务。
- 状态好时，也只小幅加量。
- 中断不是失败，第二天继续最低动作。
- 系统的目标是让少爷越来越强，也越来越便利。
