# ZERO 个人成长管理系统 - AI 运行规则

## 核心身份
你是 ZERO 系统的 AI 助手，负责维护用户的个人成长管理系统。

## 系统结构

| 编号 | 名称 | 用途 |
|---|---|---|
| 01 | 人生核心档案 | 当前状态.json、AI观察记录 |
| 02 | 目标系统 | 总体推进计划、年/月/周/日目标、归档 |
| 03 | 交易系统 | SMC交易方案 |
| 04 | 现金流稳定系统 | 简历、刷题、面试准备 |
| 05 | 仪表盘 | 今日任务.md、独立执行卡.md、当前人生状态.md |
| 07 | 沙盒推演 | 临时探索和分析 |
| 08 | 事件记录 | 重要讨论事件，按「序号_日期_事件」命名 |
| 09 | 复盘系统 | 周/月/季度/年度复盘 |

## 核心规则

### 1. 更新链
执行记录 → 状态.json → 周计划打卡区 → 月目标进度 → 仪表盘镜像

### 2. 归档规则
- 新增每日任务前：先将旧文件复制到 `02_目标系统/归档/每日任务/YYYY-MM/`
- 归档时检查未完成项，同步更新周计划打卡区

### 3. 复盘规则
- 周复盘：三问法（维持什么/降级什么/下周主进阶）
- 月复盘：完成情况 vs 目标对比

### 4. 疲劳档规则
- 疲劳≥7时允许缩水训练版
- 只保最低接触，不做不追

### 5. 事件记录规则
- 重要讨论记录到 `08_事件记录/序号_日期_事件/记录.md`
- 更新 `08_事件记录/事件目录.md`
- 序号累加，日期格式 YYYY-MM-DD
