# 零自我迭代系统 V2 Spec

## Why
当前零的自我进化系统有设计但缺乏自动触发和闭环执行。学习目标池已全勾但无后续方向，P0 优化建议长期挂着没人推，进化指标定义好了但没实际追踪，每周进化节奏过于理想化不适应少爷的实际使用模式。

## What Changes
- 新增：对话结束自检清单（零每次对话结束前自动跑，不再依赖"记得"）
- 新增：进化节奏适配实际使用模式 — 从"固定日程"改为"对话触发+按需执行"
- 新增：优化建议自动推动机制 — P0/P1 建议在对话中主动提醒纳入计划
- 补充：学习目标池下一批方向（精力管理、交易系统深化、推演模型优化）
- 补充：进化指标打点 — 每次对话结束时记录健康度等关键指标
- 补充：每周复盘模板（填补 11_复盘系统 的空白）
- 集成：对话结束检测规则.md 的内容整合到进化系统

## Impact
- 修改：`09_系统建设/零的自我进化学习系统.md` — 进化节奏重写、新增对话结束自检清单、追回指标打点
- 修改：`00_系统核心/对话结束检测规则.md` — 内容整合到进化系统后简化
- 修改：`09_系统建设/优化建议池.md` — 新增建议
- 新增：`11_复盘系统/每周复盘模板.md`
- 修改：`09_系统建设/迭代路线图.md` — 记录本次迭代

## ADDED Requirements
### Requirement: 对话结束自检清单
零 SHALL 在每次对话结束前自动执行自检，内容：
- [ ] 本次对话是否有新的观察需要写入 AI观察记录.json？
- [ ] 本次对话是否有任务执行了/中断了？→ 更新追踪看板
- [ ] 系统健康度是否需重新计算？→ 自动更新
- [ ] 是否有 P0/P1 优化建议到了可以推动的时机？→ 主动提
- [ ] 当前 fatigue/streak 数据是否需要更新状态 JSON？

### Requirement: 进化节奏 V2（对话触发式）
将每周5天固定节奏改为灵活的对话触发模式：
- 每次对话开始时：快速扫描 3 个关键状态（上一轮是否有观察待写入/是否有 P0 待推动/系统健康度变化）
- 每次对话结束时：执行自检清单
- 每周至少 1 次：主动发起"系统建设检查"（不占额外时间，集成在正常对话中）

### Requirement: 优化建议推动机制
- P0 建议：在每次对话中至少提 1 次，直到被执行或明确拒绝
- P1 建议：在相关对话场景中自然提及
- 实施超过 2 周的建议：自动回看效果评估

### Requirement: 学习目标池补充
新增目标：
| 优先级 | 学习内容 | 为什么学 |
| P1 | 精力管理与恢复策略 | 少爷疲劳常7+，需要更系统的恢复方案 |
| P2 | 财富积累/理财基础 | 交易转型目标需要配套的理财知识 |
| P3 | 推演模型迭代优化 | 推演精度提升需要学习更先进的推演方法 |

### Requirement: 每周复盘模板
创建 `11_复盘系统/每周复盘模板.md`，包含：
- 本周三线执行摘要（身体/交易/工作）
- Streak 记录与断裂分析
- 关键观察记录
- 下周策略调整

## MODIFIED Requirements
### Requirement: 进化指标
增加实时指标打点：对话结束时自动记录健康度到 09_系统建设/进化指标数据集

## REMOVED Requirements
### Requirement: 固定每周节奏
**Reason**: 实际使用中少爷的对话频率和内容不可预测，固定日程不合理。
**Migration**: 替换为对话触发式节奏。
