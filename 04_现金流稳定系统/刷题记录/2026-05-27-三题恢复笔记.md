## 1. Two Sum

### 题意一句话

给定数组和目标值，找出两个数的下标，使它们相加等于 `target`。

### 最朴素思路

双重循环，枚举两个位置。

### 暴力版最容易错的点

- 第二层循环不能从 `0` 开始，应该从 `x + 1` 开始。
- 不能让同一个元素被用两次，所以不能出现 `x == y`。
- 找到答案后直接 `return`，不要继续扫。

### 暴力版模板

```java
for (int x = 0; x < nums.length - 1; x++) {
    for (int y = x + 1; y < nums.length; y++) {
        if (nums[x] + nums[y] == target) {
            return new int[]{x, y};
        }
    }
}
```

### HashMap 版核心

不是“我去找另一个数”，而是“我先算出我缺谁，再去查之前有没有它”。

### HashMap 版模板

```java
Map<Integer, Integer> map = new HashMap<>();

for (int i = 0; i < nums.length; i++) {
    int need = target - nums[i];
    if (map.containsKey(need)) {
        return new int[]{map.get(need), i};
    }
    map.put(nums[i], i);
}
```

### 记忆点

- `containsKey`：有没有这个键
- `get`：拿这个键对应的值
- `put`：存进去
- 顺序必须是：先查，再存

## 2. Valid Parentheses

### 题意一句话

判断一串括号是否合法：类型要对，顺序也要对。

### 核心思路

用栈。

- 遇到左括号：压栈
- 遇到右括号：检查栈顶能不能配对
- 最后栈必须为空

### 最小心法

把它想成叠盘子：先放上去的，必须后拿出来。

### 最常用写法

```java
Deque<Character> st = new ArrayDeque<>();

for (int i = 0; i < s.length(); i++) {
    char c = s.charAt(i);
}
```

### 最容易手滑的点

- `Deque<Character> st = new ArrayDeque<>();`
- `char c = s.charAt(i);`
- `st.push(c);`
- `st.pop();`
- `st.isEmpty()`

### 容易错的判断

- 右括号出现时，先判断栈是不是空
- `pop` 出来的要和当前右括号类型匹配
- 扫完之后要再检查一次 `st.isEmpty()`

### 记忆点

- `push`：放进去
- `pop`：拿出来
- `isEmpty`：空不空
- `charAt(i)`：取字符串第 `i` 个字符

## 3. Best Time to Buy and Sell Stock

### 题意一句话

只能买一次、卖一次，求最大利润。

### 核心思路

一边扫数组，一边维护：

- `minPrice`：到目前为止见过的最低价
- `maxProfit`：到目前为止的最大利润

每到一天，就问一句：

```text
如果今天卖，利润是多少？
```

也就是：

```text
prices[i] - minPrice
```

### 模板

```java
int minPrice = prices[0];
int maxProfit = 0;

for (int i = 1; i < prices.length; i++) {
    if (prices[i] < minPrice) {
        minPrice = prices[i];
    } else {
        maxProfit = Math.max(maxProfit, prices[i] - minPrice);
    }
}
```

### 记忆点

- 这题不是找两天乱试
- 这题是在每一天都尝试当“卖出日”
- 卖出时，只需要知道前面最低买入价是多少
